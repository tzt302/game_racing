# 赛车 AI 强化学习步骤档案

> 当前状态：**仅完成设计与代码骨架，尚未开始训练、刷圈或生成模型。**
>
> 安全门禁：`training.train_sac` 默认只打印计划。只有人工检查后显式加入
> `--start-training` 才会创建训练环境并消耗算力。

## 1. 目标和边界

训练一个可在五条赛道自行刷圈的连续控制策略，让它从圈速和赛道反馈中学习：

- 合理的转向、循迹刹车、线性油门和刹车；
- 每个弯的入弯点、刹车起点、弯心与出弯线路；
- 在路肩可以正常驾驶，但避免长期进入缓冲区或草地；
- 用一个最佳策略派生 Easy、Normal、Hard，避免低难度只是随机乱开。

本方案首先训练单车计时赛能力。多人攻防、超车、防守和起步应作为后续独立阶段，
否则碰撞和对手行为会污染“最佳线路/刹车点”的基础学习。

## 2. 系统结构

```text
五条 Track 赛道 + Vehicle 车辆物理
              ↓ 30 Hz 控制、60 Hz 物理
       Gymnasium RacingEnv
              ↓ 连续动作/奖励/终止条件
          SAC 训练策略
              ↓ 独立评估与多随机种子复验
      最佳模型 + 圈速轨迹数据
          ↙                 ↘
  NPZ 轻量运行模型       赛车线/刹车点 JSON
          ↓
 Easy / Normal / Hard 控制器
```

离线训练依赖 Gymnasium 和 Stable-Baselines3；正式游戏只需 NumPy 加载导出的
NPZ，不需要捆绑 PyTorch。

## 3. 环境定义

### 控制频率

- 策略频率：30 Hz；
- 每次策略动作执行两个 60 Hz 车辆物理子步；
- 动作空间：`[-1, 1]³`；
- 转向：`-1` 为左满舵，`1` 为右满舵；
- 油门和刹车：分别把 `[-1, 1]` 线性映射为 `[0, 1]`；
- 刹车优先：踩刹车时按比例削减油门，避免 AI 长期同时满油满刹。

### 38 维观测

| 观测组 | 数量 | 内容 |
|---|---:|---|
| 车辆动态 | 6 | 速度、转角、侧滑角、横摆角速度、油门、刹车 |
| 车辆相对赛道 | 4 | 横向偏移、朝向误差、闭环进度 sin/cos |
| 当前路面 | 4 | 柏油、路肩、缓冲区、草地 one-hot |
| 前视信息 | 24 | 10/25/50/80/120/180 米处的曲率、课程提示速度、课程提示刹车、相对高程 |

阶段 1–2 可使用真实遥测的速度/刹车作为课程提示，帮助模型先学会稳定控车。
阶段 3–4 会把这两项置零，观测维度保持不变，但模型必须靠曲率、地形、车辆动态和
圈速反馈自行优化，不能简单复制现有理想线。

### 奖励

单步奖励由以下部分组成：

- 前向进度：主奖励，每前进 1 米奖励 `0.12`；
- 时间成本：每个控制步 `-0.025`，推动模型缩短圈速；
- 完成有效圈：`+120`，并根据相对参考圈速加减奖励；
- 动作变化：对转向/踏板突然跳变施加很小惩罚；
- 路面：路肩只施加极小成本，缓冲区和草地逐级加大惩罚；
- 倒车、长时间无进度、长时间离开赛道：额外惩罚并结束回合。

最终阶段**没有“贴近旧赛车线”的奖励**。这样发现的线路是优化结果，而不是对旧线的
模仿。参考圈只用于课程学习、对照和验收。

### 回合结束

- 完成约一整圈；
- 连续在缓冲区或草地超过 2 秒；
- 连续 5 秒没有有效前向进度；
- 达到 160 秒上限。

每个回合保存圈速、有效性、赛道限制次数、逐点横向偏移、速度、油门、刹车和路面，
供后续重建理想赛车线与刹车区。

## 4. 四阶段训练流程

| 阶段 | 默认步数 | 赛道 | 目标 | 进入下一阶段的人工门槛 |
|---|---:|---|---|---|
| 1 基础控车 | 500,000 | 蒙扎、银石 | 方向、刹停、边界；油门上限 65% | 可连续完成 20 个有效圈 |
| 2 全速课程 | 1,500,000 | 五条赛道 | 全速、换挡、不同弯型；保留遥测提示 | 各赛道有效圈率 ≥ 90% |
| 3 独立线路 | 3,000,000 | 五条赛道 | 屏蔽速度/刹车提示，自行优化线路 | 各赛道有效圈率 ≥ 95%，圈速稳定 |
| 4 精调泛化 | 2,000,000 | 五条赛道 | 严格赛道限制、随机起点和复验 | 无单赛道明显退化，选出最终模型 |

默认超参数：

- 算法：SAC；
- 网络：`256 → 256 → 128`；
- 学习率：`3e-4`；
- 回放池：1,000,000；
- batch：512；
- `gamma=0.995`，`tau=0.005`；
- 4 个并行环境；
- 每 50,000 步保存检查点并运行 10 回合确定性评估；
- 至少使用种子 `302`、`1302`、`2302` 分别复验，避免偶然得到“神圈”。

步数是初始预算，不是质量保证。每一阶段都应以验收指标而非“跑完步数”作为晋级条件。

## 5. 模型选择与防过拟合

训练模型不能仅凭单个最快圈入选。候选模型需在每条赛道完成至少 20 个有效评估圈，
并记录：

- 最快圈和中位圈速；
- 圈速标准差；
- 有效圈率和赛道限制次数；
- 同一弯的刹车点离散度；
- 是否存在蛇形转向、油刹重叠、利用物理漏洞切弯；
- 五条赛道相对上一版本是否有超过 0.5% 的退步。

建议以“中位圈速 + 无效圈惩罚 + 稳定性惩罚”排序。只有所有赛道有效圈率不低于
98%，并通过视频回放人工检查，才导出为 Hard 的基础策略。

## 6. 三档难度

三档共用经过验收的最佳路线知识，通过反应、动作频率和车辆利用率形成差异：

| 难度 | 策略更新 | 反应延迟 | 油门上限 | 转向 | 平滑/误差 |
|---|---:|---:|---:|---:|---|
| Easy | 每 4 帧 | 3 帧 | 88% | 92% | 较强平滑、少量可复现误差、略早刹车 |
| Normal | 每 2 帧 | 1 帧 | 96% | 97% | 中等平滑、极小误差 |
| Hard | 每帧 | 0 帧 | 100% | 100% | 轻度平滑、无随机误差 |

误差必须由比赛种子决定，保证重播可复现。Easy 不应故意冲出赛道；它应表现为反应稍慢、
弯速保守和更早刹车。若仅靠参数差异仍不足，再考虑分别选用较早训练检查点，但不能使用
三套互不相关的模型。

## 7. 人工执行步骤（本轮不要执行）

以下命令是将来的操作记录，不应在本轮运行。

1. 安装可选训练依赖：

   ```powershell
   python -m pip install -r requirements-rl.txt
   ```

2. 只查看计划（安全，不创建训练环境）：

   ```powershell
   python -m training.train_sac --stage 1
   ```

3. 检查奖励、回合上限、磁盘空间和硬件温度后，人工解锁阶段 1：

   ```powershell
   python -m training.train_sac --stage 1 --start-training
   ```

4. 每一阶段先检查 TensorBoard、评估视频和五赛道指标，再用上一阶段最佳检查点开始下一阶段：

   ```powershell
   python -m training.train_sac --stage 2 --resume training/checkpoints/stage_1/best/best_model.zip --start-training
   ```

5. 阶段 4 完成后，对最终候选模型执行独立、确定性评估；不要直接采用最后一个 checkpoint。

6. 人工确认模型后，导出正式游戏可加载的 NumPy 权重：

   ```powershell
   python -m training.export_policy MODEL.zip training/exports/elite_policy.npz --confirm-export
   ```

7. 对多圈有效轨迹求中位数，生成学习到的赛车线、速度曲线和刹车点：

   ```powershell
   python -m training.extract_racecraft trajectory.json learned_racecraft.json --confirm-extract
   ```

8. 最后再把策略接入正式比赛 AI。接入前保留当前规则式 AI 作为回退；权重不存在、校验失败
   或推理异常时必须自动使用旧 AI，不能让游戏无法启动。

- [ ] 已保存上一版本和训练配置；
- [ ] 确认不会覆盖正式游戏资源；
- [ ] 检查可用磁盘、内存和显存；
- [ ] 固定训练/评估随机种子；
- [ ] 先用极小步数做独立冒烟测试；
- [ ] 确认轨迹没有反向、跳点或跨越起终点错误；
- [ ] 确认路肩可用、缓冲区/草地惩罚符合预期；
- [ ] 训练期间只写入 `training/runs` 和 `training/checkpoints`；
- [ ] 达到阶段门槛并人工观看回放后才晋级。

## 9. 已创建的代码入口

- `training/racing_env.py`：环境、观测、动作、奖励和轨迹记录；
- `training/train_sac.py`：带显式安全门禁的四阶段 SAC 训练入口；
- `training/export_policy.py`：把 SAC actor 导出为轻量 NPZ；
- `training/extract_racecraft.py`：从有效轨迹提取线路和刹车数据；
- `src/ai/rl_policy.py`：正式游戏可使用的 NumPy 推理和难度控制器；
- `requirements-rl.txt`：可选训练依赖，未加入普通游戏依赖。

正式接入比赛循环被有意推迟：在没有经过验收的权重前，不改变当前玩家可运行的 AI。
